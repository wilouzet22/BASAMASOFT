Parámetros estructurados para generación de código de alto nivel

1. Levantamiento exhaustivo de requerimientos
La IA debe formular todas las preguntas necesarias antes de generar cualquier línea de código.
Debe validar:

Objetivo del sitio web
Público objetivo
Funcionalidades clave
Restricciones técnicas
Requerimientos legales o de seguridad

👉 Si la IA no pregunta lo suficiente, está trabajando a ciegas. Y un sistema a ciegas en producción… ya sabes cómo termina.

2. Estándares de calidad para entorno productivo
El código generado debe cumplir con estándares reales de producción:

Código limpio, modular y escalable
Buenas prácticas (SOLID, DRY, KISS)
Manejo de errores robusto
Seguridad (validación de datos, protección contra vulnerabilidades)
Optimización de rendimiento

3. Diseño de Interfaces de Usuario (UI) de nivel profesional
La IA debe generar interfaces:

Visualmente coherentes y modernas
Responsivas (adaptables a cualquier dispositivo)
Con jerarquía visual clara
Uso adecuado de tipografía, colores y espaciado

👉 Nada de interfaces “bonitas pero inútiles”.

4. Experiencia de Usuario (UX) optimizada
El sistema debe priorizar:

Navegación intuitiva
Flujo lógico de interacción
Reducción de fricción del usuario
Accesibilidad (WCAG básico al menos)
Tiempos de carga rápidos

👉 Si el usuario se pierde, el proyecto fracasó. Así de simple.

5. Coherencia con la identidad institucional
El sitio debe reflejar fielmente:

Misión, visión y valores de la empresa
Manual de marca (colores, logos, tipografía)
Tono comunicativo institucional
Consistencia visual en todas las secciones
⚠️ Regla implícita (pero crítica)

La IA no debe generar código hasta cumplir completamente los 5 puntos anteriores.
